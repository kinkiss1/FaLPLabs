# Генератор писем DOCX

Минимальное WPF-приложение для лабораторной работы.

## Что делает программа

1. Показывает форму ввода данных письма.
2. Использует готовый DOCX-шаблон `Templates/Template.docx` с плейсхолдерами.
3. Копирует шаблон в новый файл `GeneratedLetter.docx`.
4. Заменяет плейсхолдеры значениями из формы через `DocumentFormat.OpenXml`.
5. Выполняет замену не только в тексте документа, но и в колонтитулах.
6. Добавляет одно или несколько приложений к письму.

## Плейсхолдеры

```text
{DATE}
{RECIPIENT_POST}
{RECIPIENT_NAME}
{SUBJECT}
{LETTER_BODY}
{SENDER_NAME}
{APPENDIX_LIST}
```

## Приложения

В поле `Приложения` каждое приложение вводится с новой строки:

```text
Заголовок | количество листов | текст приложения
```

Пример:

```text
Причина отмены лабораторной | 3 | Здесь находится текст первого приложения.
Почему всё плохо | 2 | Здесь находится текст второго приложения.
```

В письмо автоматически добавляется перечень приложений, а сами приложения вставляются отдельными страницами после письма.

## Запуск

```powershell
dotnet run
```

Шаблон письма находится здесь:

```text
Templates\Template.docx
```

В шаблоне есть верхний колонтитул с датой и темой письма, а также нижний колонтитул с отправителем.

После нажатия кнопки `Сформировать DOCX` результат появится в папке сборки:

```text
bin\Debug\net8.0-windows\GeneratedLetter.docx
```
либо
```text
bin\Release\net8.0-windows\GeneratedLetter.docx
```
