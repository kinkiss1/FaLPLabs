# Генератор писем DOCX

Минимальное WPF-приложение для лабораторной работы.

## Что делает программа

1. Показывает форму ввода данных письма.
2. Использует готовый DOCX-шаблон `Templates/Template.docx` с плейсхолдерами.
3. Копирует шаблон в новый файл `GeneratedLetter.docx`.
4. Заменяет плейсхолдеры значениями из формы через `DocumentFormat.OpenXml`.
5. Выполняет замену не только в тексте документа, но и в колонтитулах.
6. Добавляет одно или несколько приложений к письму.

## Плейсхолдеры

```text
{DATE}
{RECIPIENT_POST}
{RECIPIENT_NAME}
{SUBJECT}
{LETTER_BODY}
{SENDER_NAME}
{APPENDIX_LIST}
```

## Что реализовано

1. Форма ввода данных письма.
2. DOCX-шаблон `Templates/Template.docx` с плейсхолдерами.
3. Замена плейсхолдеров в тексте документа и колонтитулах.
4. Добавление приложений к письму.
5. Базовая валидация обязательных полей.

## Третья лабораторная

В третьей лабораторной добавлено:

- поля `ФИО адресата`, `Тема письма`, `Текст письма`, `ФИО отправителя` помечены звёздочкой как обязательные;
- внизу формы добавлена сноска: `* — поля обязательные к заполнению`;
- если обязательные поля не заполнены, появляется предупреждение и документ не создаётся;
- элементы формы разделены на отдельные блоки;
- приложения добавляются через кнопку `+ Добавить приложение`;
- каждое приложение заполняется в отдельной карточке.

## Запуск

```powershell
dotnet run
```
Шаблон письма находится здесь:

`Templates\Template.docx`

В шаблоне есть верхний колонтитул с датой и темой письма, а также нижний колонтитул с отправителем.
После нажатия кнопки `Сформировать DOCX` файл появится в папке сборки:

```text
bin\Debug\net8.0-windows\GeneratedLetter.docx
```
либо
```text
bin\Release\net8.0-windows\GeneratedLetter.docx
```
